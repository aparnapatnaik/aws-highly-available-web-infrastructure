# AWS Highly Available Web Infrastructure

##  Project Overview

This project demonstrates the deployment of a highly available web infrastructure on Amazon Web Services (AWS). It uses multiple EC2 instances behind an Application Load Balancer (ALB), an Auto Scaling Group (ASG), and VPC Peering for secure communication between two Virtual Private Clouds (VPCs).

---

##  AWS Services Used

- Amazon EC2
- Amazon VPC
- Public Subnets
- Internet Gateway
- Route Tables
- Security Groups
- Application Load Balancer (ALB)
- Target Group
- Launch Template
- Auto Scaling Group (ASG)
- VPC Peering

---

##  Architecture

The project consists of:

- VPC-A
- VPC-B
- Public Subnets
- Internet Gateway
- EC2 WebServer-1
- EC2 WebServer-2
- Application Load Balancer
- Target Group
- Launch Template
- Auto Scaling Group
- VPC Peering Connection

---

##  Features

- High Availability using Application Load Balancer
- Automatic Instance Scaling using Auto Scaling Group
- Apache Web Server deployment
- Secure networking using Security Groups
- Private communication through VPC Peering
- Health Checks using Target Groups
